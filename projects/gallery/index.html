Here's how the main functionality is done

Not all images are uploaded in the same time. Only miniatures and the first full size one are processed at start.

Others — the ones that you can see only after clicking on a miniature — are uploaded later. That way they don't "steal bandwith" and the gallery appears faster.

Before the first full size photo is uploaded, it's miniature version serves as placeholder. That way you shorten blank space appearance.

If you're a developer, check out the source code (it's un-minified). With the dev tools' network tab you can also see differences in image upload time. It will be useful to use 2G simulation and disable cache. Notice that images '2' and '3' are uploading after miniatures and '1' are done.

For aesthetic reasons I used also CSS animations and filters (blur).
